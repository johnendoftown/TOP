Knowledge Check: Commit Messages

What are two benefits of having well-written commit messages and a good commit history?

To be able to clearly collaborate with others and refer to your history of changes

How many characters should the subject line of your commit message be?

50, with a maximum of 72

Notes

Using the command line to write Git commits is preferable

Commits use the imperative mood, which is spoken as if given a command or instruction. To do this clearly, use the phrase "If applied, this commit will...". For example:

  • Update getting started documentation
  • Release veriosn 1.0

The title should be capitalized and no more than 50 characters, more will be cut off by github
